# 为什么本地代码修改后，网页却没有变化？

您遇到的情况是非常经典的“后台没重启”或者“浏览器缓存”问题。因为 Python (Flask) 默认情况下，一旦启动，它就会把旧代码加载到内存中。就算您后来用编辑器（或者我帮您）修改了 `app.py` 和各种 `html`，**只要没有杀掉旧的 Flask 进程并重新运行，网页上跑的永远是修改前的老代码！**

我已经刚刚在后台帮您强制杀掉了所有旧的 `flask` 进程，并且重新启动了它。

👉 **请现在立刻做以下两步：**

1. 回到您的浏览器，**不要点击普通的刷新**。
2. 按住键盘上的 **`Ctrl`** 键不放，同时按下 **`F5`** 键（这叫做强制刷新，会无视缓存重新拉取最新的 HTML 和资源）。
   - *(如果在 Mac 上，请按 `Cmd + Shift + R`)*

您现在应该能看到导航栏里多出了闪闪发光的 **AI菜单**，首页也多出了 **冰箱贴** 板块！

---

### 如果以后再遇到类似问题，请记住以下操作步骤：

在您本地测试时（尤其是在 Windows 的 VSCode / Pycharm 终端里）：
1. 先在运行着程序的黑框框（终端）里按下 **`Ctrl + C`** 结束当前服务。
2. （安全起见）确保没有旧的进程阻挡：
   ```powershell
   Get-Process -Name "python", "flask" | Stop-Process -Force -ErrorAction SilentlyContinue
   ```
3. 重新输入启动命令：
   ```powershell
   flask run
   ```
4. 去浏览器 **强制刷新（Ctrl + F5）**。
